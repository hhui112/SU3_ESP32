# SU2 V2.0 双传感器协议答疑

> 依据：《双_智能床通信协议20260604.docx》V2.0  
> 用途：回复传感器/协议方疑问，并记录上位机侧结论与待确认项

---

## V2.0 核心变化（摘要）

| 项目 | 约定 |
|------|------|
| 传感器1（左 L） | `src_addr / dest_addr = 0x03` |
| 传感器2（右 R） | `src_addr / dest_addr = 0x06` |
| 新增 topic | **16 = 1 秒实时数据** |
| report / list | 按 `dest_addr` 定向下发；回包保留 `src_addr`；多帧**同源、串行、禁止交织** |

## 1. 1 秒数据 Sign

**文档：** 4 字节，仅定义 Sign[0~2]。

| 字节 | 含义 |
|------|------|
| Sign[0] | =1：两侧信号相似（>50%） |
| Sign[1] | 0~100：相似度百分比 |
| Sign[2] | 0：本路为偏弱侧；1：偏强侧 |

**答复要点：**

- Sign 表示**单人睡双人床、双侧检出**的融合信息，**不是左右编号**。
- **左右侧用帧头 `src_addr` 区分**：03=左/传感器1，06=右/传感器2。
- Sign[2] 表示**当前这路**偏弱/偏强，与 `src_addr` 组合理解。
- **Sign[3] 未定义**，请协议方明确（Reserved 或 bit 域）；5 秒数据 Sign 为 15 字节，也仅定义 [0~2]，其余字节需补全。

## 2. 1 分钟数据 SBP / DBP

**文档：** topic 14 仍含 SBP/DBP；CLI 仍有 `set/get sbp/dbp/bpn`；report 仍有 topic 11/12。

**答复要点（若已取消动态血压）：**

| 层级 | 建议 |
|------|------|
| 实时 1min（topic 14） | 字段保留但填 0/0xFF，或标 Deprecated |
| CLI `set/get sbp/dbp/bpn` | 标废弃或固定返回 |
| 睡眠报告 topic 11/12 | 明确是否仍发送；若不发送，修改 `report x` 说明 |

**待确认：** V2.0 是否仍输出 report 11/12。

## 3. 睡眠呼吸暂停流水号（topic 15）

**文档：** `sequence` 4 字节，**未规定范围**；`status_flag`：1=发生，0=结束。

**答复要点：**

- 类型：**uint32**，0 ~ 4294967295，溢出回绕。
- 计数：**按传感器（03/06）各自递增**。
- 建议配对规则：**发生（flag=1）时 sequence+1，结束（flag=0）沿用同一 sequence**（需传感器确认）。

## 4. hello 命令

**文档：** V2.0 仍保留 hello（100ms，src/dest=0），与“拟取消”不一致。

**答复要点：**

- **不建议取消**：CLI 需 24 字节 device_id；上电后需 `set rtc` 等，取消 hello 则只能等 2 分钟进工作态后才能从数据获知 ID。
- **折中：** 保留 hello；间隔改为 **500~1000ms**；payload 带 **device_id**；双传感器 hello 用 **src_addr 03/06** 区分。
- 若坚持取消，协议须补充：**未进工作态前如何获取 device_id**。

## 6. set mode 1 / 2

**文档矛盾：** mode 2 仍写“5 秒+1 分钟”，但新增 topic 16 为 **1 秒** 数据。

**答复要点：**

| 命令 | 建议含义 |
|------|----------|
| set mode 0 | 停止，空闲 |
| set mode 1 | **仅 1min（topic 14）**，关 1s |
| set mode 2 | **1s（topic 16）+ 1min（topic 14）**，默认 |
| set mode 3 | 固件升级 |
| set mode 4 | 立即工作 |

- **关 1 秒、仅 1 分钟 → 用 set mode 1**，理解正确。
- **待确认：** topic 13（5 秒）在 V2.0 是否废弃。

## 7. get mode 返回值

**文档：** 0=空闲；1=1min；2=5s+1min；3=升级；4=工作中；5~10=故障；**有故障优先返故障**。

**答复要点：**

- **1/2 = 配置态**（已设模式）；**4 = 运行态**（正在采集）— 语义重叠，需协议二选一：
  - **方案 A（推荐）：** 无故障只返 **1 或 2**，废弃单独返 4。
  - **方案 B：** 工作中只返 **4**，1/2 仅在 `get param` 体现配置。
- **9 LSE：** 外部低速晶振（RTC）故障 → 对时/时间戳不可靠。
- **10 HSE：** 外部高速晶振故障 → 主时钟/采样异常。

## 8. 校准指令

**文档：** `set calibration manual/auto`（手动/自动校准）；`set sbp/dbp/bpn`（血压参数）。

**答复要点：**

- **手动校准命令仍在**：`set calibration manual` / `auto`，与 sbp 类命令不同。
- 若取消动态血压：**保留 calibration**；**废弃 sbp/dbp/bpn**。
- **待补充：** 手动校准完整流程及校准对象（体动/压力阈值等）。

**文档：** 新增 Sign 4 字节，**内容待定**。
